Project Task Dashboard

AWS Cloud Security Enhancement Project

1. Environment Setup

Objective: Establish a functional AWS test environment by 31st May 2025

Sub-Tasks:
  • AWS Account Creation: Register free-tier AWS account for EC2, S3, RDS.
  • VPC Configuration: Create VPC with public subnet (CIDR 10.0.1.0/24).
  • EC2 Instance Setup: Launch t2.micro instance with SSH/HTTP access.
  • S3 Bucket Creation: Create bucket with versioning enabled.
  • RDS Instance Setup: Deploy MySQL RDS instance (db.t2.micro).
  • Verification: SSH to EC2, upload file to S3, connect to RDS.

Deliverable: Functional AWS test environment, verified by SSH and S3 upload, by 31st May 2025.

Contingency: Terminate unused resources or use AWS Educate credits if Free Tier limits reached.

Technical Details: Aligns with NIST SP 800-53 (SC-7) for network segmentation.

2. Data Loss Prevention Setup

Objective: Implement AWS Macie for S3 scanning by 14th June 2025

Sub-Tasks:
  • Macie Activation: Enable Macie and link to S3 bucket.
  • S3 Scan Configuration: Scan for basic PII using default identifiers.
  • IAM Role Setup: Create role with S3 read-only access, apply bucket policy.
  • Verification: Review Macie scan results in console.

Deliverable: DLP configuration report, verified by Macie, by 14th June 2025.

Contingency: Manually check S3 permissions if Macie fails.

Technical Details: Meets GDPR Article 32 for data protection.

3. Encryption Implementation

Objective: Configure encryption for S3, RDS, EC2 by 5th July 2025

Sub-Tasks:
  • S3 Encryption: Enable AES-256 encryption on bucket.
  • RDS Encryption: Enable encryption at rest with default KMS key.
  • HTTPS Configuration: Provision TLS certificate for EC2 web server.
  • Verification: Confirm encryption with CLI and OpenSSL.

Deliverable: Encrypted S3, RDS, and HTTPS-enabled EC2, tested by 5th July 2025.

Contingency: Use default S3 encryption or self-signed certificate if needed.

Technical Details: Aligns with NIST SP 800-53 (SC-28) and GDPR.

4. DDoS Protection Configuration

Objective: Enable AWS Shield and WAF by 26th July 2025

Sub-Tasks:
  • Shield Standard Activation: Enable for VPC resources.
  • WAF Configuration: Create web ACL with rate-limiting rule.
  • Attack Simulation: Simulate HTTP flood with JMeter.
  • Verification: Review CloudWatch logs for threat handling.

Deliverable: Protected environment, confirmed by JMeter, by 26th July 2025.

Contingency: Use Shield Standard if WAF fails.

Technical Details: Simulates Layer 7 attacks per NIST SP 800-53 (SC-5).

5. Penetration Testing

Objective: Conduct penetration testing by 16th August 2025

Sub-Tasks:
  • AWS Approval: Submit security approval by 12th July 2025.
  • Network Scanning: Run Nmap on EC2 for open ports.
  • Web Vulnerability Testing: Use OWASP ZAP for basic scans.
  • Verification: Document and fix vulnerabilities, re-scan.

Deliverable: Vulnerability report with fixes, by 16th August 2025.

Contingency: Use AWS Trusted Advisor or manual checks if tests fail.

Technical Details: Aligns with NIST SP 800-115 for minimal resource use.
